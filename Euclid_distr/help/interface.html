Описание интерфейса пользователя.

Основное диалоговое окно программы.

Основное диалоговое окно представлено на рис.1.


Рис.1. Основное диалоговое окно.

1. . Если многочлен g(x) задан, то при нажатии на эту кнопку будет произведена проверка заданного g(x) на наприводимость. В случае, если многочлен не прошел тест на неприводимость, будет выведено сообщение об ошибке, сообщающее, что чтобы построить расширение простого поля, многочлен g(x) должен быть неприводимым. Если g(x) не задан, то программа осуществит подбор нормированного неприводимого многочлена. Первый многочлен, удовлетворяющий условиям неприводимости над указанным полем, будет записан в поле ввода для g(x)(11).

2. Поле ввода "Module" - указывает основание простого поля. Попытка начать вычисления при пустом поле "Module", закончится неудачей - программа выведет сообщение об ошибке. Если в качестве "module" задать что-либо отличное от числа или число, меньшее 1, то также будет выведено соответствующее сообщение об ошибке. Если "Module" задать равным 1, то программа не выведет никаких сообщений об ошибке и будет проводить вычисления, но результаты этих вычислений будут абсурдны (например можно получить 0 в качестве частного двух многочленов, отличных от нуля).

    Замечание: Программа не производит проверку указанного модуля на простоту, поэтому забота о том, чтобы в поле было введено простое число возлагается на пользователя программы. Если модуль задать не простым, то вычисления будут проводиться, но, как нетрудо догадаться они не будут иметь смысла (более того, можно получить системную ошибку "делние на ноль").

3. . Поиск примитивных и непрводимых многочленов заданной степени. При нажатии на эту кнопку появится окно, изображенное на рис. 2.


Рис. 2. Окно для поиска примитивных и непрводимых многочленов.

    f1. Поле ввода для степени многочлена. Если в этом поле указать что-либо отличное от натурального числа, появится сообщение об ошибке.

    f2. Поле вывода для результата.

    f3. . Если расчет был начат, то при нажатии на эту кнопку он будет прерван, а диалоговое окно, изображенное на рис. 2 будет закрыто. Если расчет начат не был, то диалоговое окно рис. 2 просто закроется, аналогично нажатию кнопки [X].

    f4. . Кнопка начала расчета.

    f5. "Find All Irreducible". Если этот флаг выбран, то будет произведен поиск неприводимых нормированных многочленов. В отсутствие флага (по умолчанию) будет произведен поиск примитивных многочленов.

4. "big-endian". Если этот флаг поставлен, то программа будет считать, что все исходные многочлены, задаваемые пользователем указаны в "тупоконечном" формате, т.е. первыми (слева направо) выводятся коэффициенты при старших степенях аргумента. При этом, все выводимые программой в качестве результатов многочлены, тоже будут записаны в "тупоконечном" формате. Если флаг не поставлен (по умолчанию), то весь ввод и вывод программа будет производить в "остроконечном" формате, т.е. первыми (слева) будут записываться коэффициенты при младших степенях аргумента.

    Замечание: в файле регистрации действий запись многочленов производится только в "остроконечном" формате.

5. "no log". Если указан этот флаг, то программа не ведет файл регистрации. Эта возможность может значительно увеличить скорость расчетов, что немаловажно при больших значениях module, а также при поиске примитивных и неприводимых многочленов больших степенией.

Назавание файла регистрации формируется следующим образом: poly_integer.log, где integer представляет собой число, равное интервалу времени между запуском программы и полуночью 1 января 1970 г, выраженном в секундах.

6. . Кнопка, позволющая загрузить многочлен #1 из файла.

7. Шкала индикации, отображающая текущую стадию вычисления элементарной операции над многочленами.

    Замечание: Элементарными операциями считаеются [+],[-] и все логические операции сравнения многочленов. На основе этих операций строятся более сложные, например, [*], [/], алгоритм Евклида и прочие.

8. Главная шкала индикации, отображающая текущую стадию вычисления комплексной операции, например, поиска примитивных многочленов.

    Замечание: 1. Главная шкала индикации не отображает стадии вычисления элементарных операций, так как они достаточно быстро вычисляются.
    2. Главная шкала индикации дает лишь приблизительную оценку оставшегося количества вычислений и не должна являться основополагающей при оценке времени вычислений. Более того, для больших степеней полиномов, главная шкала индикации может "ошибаться".
    3. Между шкалами индикации расположена строка состояния, информирующая пользователя о характере проводимых вычислений. По окончании вычислений в строке состояния отобразится время, потребовавшееся пограмме для проведения последних вычислений с точностью до секунды.

9. . Поиск обратного для многочлена #1 в поле Fp[x]/g(x) (если задан g(x)) или реализация расширенного алгоритма Евклида для многочленов #1 и #2 (если g(x) не задан).

10. . Деление многочлена #1 на #2. Если задан g(x), то деление производится в поле Fp[x]/g(x)

11. Поле ввода g(x). Если g(x) не задан, то при нажатии на в это поле будет подставлен неприводимый многочлен заданной степени. Многочлен можно загрузить из файла, нажатием кнопки , рсположенной справа.

12. #1. Поле ввода для первого многочлена-операнда. Многочлен можно загрузить из файла, нажатием кнопки , рсположенной справа.

13. #2. Поле ввода для второго многочлена-операнда. Многочлен можно загрузить из файла, нажатием кнопки , рсположенной справа.

14. res. Поле вывода результата вычислений. Многочлен результата можно сохранить в файле, нажатием кнопки , расположенной справа.

15. . При нажатии на эту кнопку все поля ввода будут очищены.

16. . Позволяет считать многочлен #1 из файла.

17. . Позволяет считать многочлен #2 из файла.

18. . Позволяет сохранить многочлен res в файле.

19. . Выполнение операции вычитания многочлена #2 из #1. Результат помещается в поле res. Если задан g(x), то результат приводится по модулю g(x).

20. . Выполнение операции умножения многочлена #1 на #2. Результат помещается в поле res. Если задан g(x), то результат приводится по модулю g(x)

21. . Кнопка, позволяющая прервать затянувшиеся вычисления.

22. . Выполнение операции сложения многочленов #1 и #2. Результат помещается в поле res. Если задан g(x), то результат приводится по модулю g(x)
